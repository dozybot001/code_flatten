# 📦 Project Packer

**让 AI 更懂你的代码项目。**

Project Packer 是一个专为 AI 辅助编程设计的轻量级工具。它解决了将本地项目上下文投喂给 LLM（如 ChatGPT, Claude, Gemini）时繁琐的“复制粘贴”痛点，并支持将 AI 生成的代码一键还原为文件结构。

### 🌟 核心功能

  * **🚀 智能打包 (Pack):** 自动过滤 `node_modules`, `.git`, 图片等干扰文件，提取核心代码。
  * **🌳 结构可视化:** 生成清晰的 `Project Structure` 树形图，帮助 AI 理解文件依赖关系。
  * **🔄 极速还原 (Unpack):** 能够解析包含特定标记的文本，一键生成项目 ZIP 包，无需手动新建文件。
  * **📊 Token 估算:** 实时显示打包后的 Token 数量，防止超出 AI 上下文限制。
  * **🔒 隐私安全:** 纯前端实现（HTML/JS），所有文件处理均在**本地浏览器**完成，不会上传到服务器。

-----

### 💡 使用场景 (Use Cases)

Project Packer 不仅仅是一个合并工具，它适用于多种开发流：

**1. 代码重构与优化**

> "帮我重构这个模块的代码..."

  * 将整个功能模块文件夹打包，让 AI 拥有完整的上下文，避免因缺少引用关系导致的“幻觉”代码。

**2. 新功能开发 (Feature Implementation)**

> "基于现有的项目结构，帮我添加一个用户登录页面..."

  * 投喂现有的 API 定义和路由结构，让 AI 生成完美契合现有风格的新代码。

**3. 跨语言/框架迁移**

> "将这个 React 组件改写为 Vue..."

  * 打包旧项目代码，要求 AI 输出新框架的代码包，利用还原功能快速建立新项目原型。

**4. Debug 与 错误排查**

> "这个项目运行报错了，这是代码结构，请分析原因..."

  * 提供完整的项目全景图，帮助 AI 快速定位文件路径错误或循环依赖问题。

**5. 快速阅读开源代码**

  * 将下载的 GitHub 开源项目打包成一个文本文件，直接发给 AI 让其“解释代码逻辑”，作为阅读向导。

### 📖 如何使用

#### 📂 模式 A：打包 (Pack)

1.  点击上传或**拖拽**项目文件夹到左侧区域。
2.  工具会自动过滤无关文件（如 `bin`, `dist` 等）。
3.  在列表中勾选/取消特定文件。
4.  点击 **"复制到剪贴板"**，直接粘贴给 AI。

#### 📦 模式 B：还原 (Unpack)

1.  在与 AI 对话时，使用下方的**推荐 Prompt**。
2.  将 AI 生成的完整代码块（包含 `=== File: ... ===` 标记）复制。
3.  粘贴到 Project Packer 的还原区域。
4.  点击 **"解析并下载 Zip"**，解压即可使用。

-----

### 📝 推荐 Prompt (提示词)

为了确保 AI 输出的内容能被本工具完美还原，请在提问时附带以下要求：

```text
请修改/生成代码，并以 Project Packer 格式（包含 Project Structure 和 === File: path === 标记）输出完整的修改后文件内容，不要省略任何代码。
```
